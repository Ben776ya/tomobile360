# Audits

Read-only data-quality audits. Nothing here writes to the database, and nothing
auto-corrects — each audit produces a list a human reviews before acting.

## category-suspects

Flags `models.category` values that look inconsistent (missing, non-standard, or
a well-known nameplate in the wrong body style).

- **Query:** [`category-suspects.sql`](./category-suspects.sql) — SELECT-only.
- **Output:** [`category-suspects.csv`](./category-suspects.csv) — columns
  `brand, model, current_category, reason, confidence`.

### Producing the CSV

The CSV currently holds only its header row. Live database access was not
authorized in the session that added this audit, so the data rows must be
generated by an operator:

1. Open the Supabase SQL editor against the **production** database.
2. Run `category-suspects.sql`.
3. Export the result grid as CSV and overwrite `category-suspects.csv`.

### Acting on results

Review each row. If a category is genuinely wrong, correct it in `/admin`
(or via a deliberate, reviewed migration). Do **not** bulk-apply the heuristic —
`confidence = medium` rows in particular are prompts to look, not verdicts.
