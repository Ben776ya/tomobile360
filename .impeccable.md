# Tomobile 360 — Design Context

This file is the source of truth for design decisions. It is consumed by the Impeccable design skills (`/audit`, `/normalize`, `/polish`, etc.) and should be read at the start of any design work.

---

## Design Context

### Users

- **Primary — Moroccan car buyers (mobile-first).** Browse `/`, `/neuf`, `/occasion`, `/actu`, `/videos`, `/forum`. Most traffic is on phones, mixed tech literacy, French-language. Their job: find a car, compare specs, contact a seller, read news.
- **Secondary — Sellers and dealerships.** Post used listings via `/occasion/vendre`, manage their own listings in `/compte`, and (for dealers/admins) operate the `/admin/*` panel for vehicles, articles, promotions, imports, video sync.
- **Tertiary — Enthusiasts.** Use the forum and video gallery; lower priority for design decisions.

### Brand Personality

- **Three words:** trustworthy, fast, scannable.
- **Voice:** professional, helpful, direct. French copy should be neutral-formal, not playful or jargon-heavy.
- **Emotional goals:** confidence and clarity — users should feel they can find what they need quickly and trust what they see. Not delight, not wow, not luxury.
- **Reference posture:** AutoTrader, Cars.com, leboncoin — modern marketplace utility, blue + navy, utilitarian.

### Aesthetic Direction

- **Theme:** light primary surfaces for content; dark navy (`#1C2541`) reserved for header, footer, admin chrome, and the occasional dark-themed promo banner. No user-toggleable dark mode.
- **Color core:** navy (`#1C2541`) as anchor + tech blue (`#006EFE`) as accent + neutral grays + standard semantic colors (success/warning/destructive). Drop the unused "champagne gold" CSS variable and "Maybach Two-Tone" naming — they conflict with the marketplace direction.
- **Type:** Sora (body, `font-sans`) + Montserrat Alternates (display, `font-display`). Already loaded via `next/font` in the root layout — keep.
- **Anti-references / what this should NOT look like:**
  - Futuristic / cyberpunk dashboards (neon glows on dark backgrounds, glassmorphism, animated gradients)
  - Hero metric layouts (giant number + small label + sparkline)
  - Identical card grids with icon + heading + paragraph repeated everywhere
  - Gradient text on headings or stats
  - The current `gradient-futuristic`, `glow-pulse`, `shimmer`, `float` animations applied as decoration on persistent elements
- **Accessibility:** WCAG 2.1 AA minimum across all consumer and admin surfaces. 4.5:1 body contrast, real labels (not placeholder-as-label), visible focus indicators, ≥44×44px touch targets.

### Design Principles

1. **Trust over flash.** A marketplace lives on credibility. Clarity, contrast, and predictable patterns always beat decorative animation. If it looks like a startup pitch deck, it's wrong.
2. **Mobile-first scan-ability.** Card grids must read at a glance on a 360px viewport. Hierarchy, image quality, and price legibility come before visual flourish.
3. **Tokens are law.** Every color, radius, and shadow comes from `tailwind.config.ts` or the CSS variables in `globals.css`. No hard-coded hex. Conflicting tokens (the gold variable that nobody renders) get removed, not preserved "just in case".
4. **Motion is for state, not decoration.** Animate entrances, exits, and feedback. Do not animate idle elements. Drop `glow-pulse`, `shimmer`, and `float` from persistent UI; keep them only for genuine loading states.
5. **WCAG AA is a build requirement.** Contrast is verified on real combinations (gray-on-color is the common failure); keyboard navigation works without a mouse; focus rings are always visible.

## How to Use This File

- Design skills (`/audit`, `/normalize`, `/polish`, `/clarify`, `/distill`, etc.) read this section before doing any work.
- If a principle here conflicts with what's in code, the principle wins — flag it as drift in the audit, then fix the code.
- Update this file when the brand intentionally evolves; do not let the code's accumulated patterns silently redefine the direction.
